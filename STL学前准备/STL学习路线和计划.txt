STL学习路线和计划

适合对象：
已经学过C++基础语法，知道变量、函数、类、引用、指针、模板基础，但对STL还不系统的人。

总目标：
1. 能熟练选择合适的容器。
2. 能正确使用迭代器遍历和修改数据。
3. 能熟练使用常用算法，少写重复循环。
4. 能理解容器、迭代器、算法、函数对象之间的关系。
5. 能用STL完成常见数据处理、查找、排序、统计、模拟、刷题和小项目。


一、学习STL之前要先补好的C++基础

1. 模板基础
   - 函数模板：template <typename T>
   - 类模板：vector<int>、map<string, int>这类写法的含义
   - typename、auto、decltype的基本使用

2. 引用和const
   - T&、const T&、T&&的基本区别
   - 为什么遍历容器时经常写 const auto& x
   - 函数参数什么时候传值，什么时候传引用

3. 运算符重载
   - operator<
   - operator==
   - 自定义类型放进set、map、priority_queue时为什么经常要定义比较规则

4. lambda表达式
   - [] {}
   - [=]、[&] 捕获
   - [](int a, int b) { return a < b; }
   - lambda在sort、find_if、count_if中的使用

5. 基础内存概念
   - 栈和堆的区别
   - vector扩容为什么会导致迭代器失效
   - 对象拷贝、移动的基本概念


二、STL核心学习路线

第1阶段：认识STL整体结构

要学什么：
1. STL由哪些部分组成
   - 容器：vector、list、map等，用来存数据
   - 迭代器：begin、end，用来访问数据
   - 算法：sort、find、count、reverse等，用来处理数据
   - 函数对象：less、greater、lambda，用来定义行为
   - 适配器：stack、queue、priority_queue，用特殊接口包装容器
   - 分配器：allocator，负责底层内存管理，初学了解即可

2. 理解“容器 + 迭代器 + 算法”的配合方式
   示例：
   vector<int> v = {3, 1, 2};
   sort(v.begin(), v.end());

重点：
STL不是一堆孤立工具，而是一套“数据结构 + 通用算法”的组合系统。

练习：
1. 创建vector<int>，插入10个数。
2. 使用sort排序。
3. 使用find查找某个数字。
4. 使用for、迭代器、范围for分别遍历。


第2阶段：顺序容器

要学什么：
1. vector
   - push_back、pop_back
   - size、empty、clear
   - operator[]、at
   - front、back
   - insert、erase
   - reserve、capacity
   - vector扩容和迭代器失效

2. deque
   - push_front、push_back
   - pop_front、pop_back
   - 适合两端频繁插入删除

3. list
   - 双向链表
   - insert、erase效率高
   - 不支持随机访问，不能写 list[i]
   - list自己的sort、merge、splice

4. forward_list
   - 单向链表
   - 初学了解即可

5. array
   - 固定大小数组
   - array<int, 5>
   - 比普通数组更安全，支持size、begin、end

重点比较：
1. vector：最常用，随机访问快，尾部插入快。
2. deque：两端插入删除快。
3. list：中间插入删除快，但查找慢，不支持下标。
4. array：大小固定，适合固定长度数据。

练习：
1. 用vector保存学生成绩，求最高分、最低分、平均分。
2. 用deque模拟排队，支持队首和队尾加入。
3. 用list保存任务列表，支持在中间插入和删除任务。
4. 比较vector和list查找某个元素的写法差异。


第3阶段：关联容器

要学什么：
1. set
   - 自动排序
   - 元素唯一
   - insert、erase、find、count

2. multiset
   - 自动排序
   - 元素可以重复

3. map
   - key-value结构
   - key自动排序且唯一
   - insert
   - operator[]
   - find
   - erase

4. multimap
   - key可以重复
   - 初学了解即可

5. unordered_set
   - 哈希表
   - 查找平均O(1)
   - 不自动排序

6. unordered_map
   - 哈希表版map
   - 统计频率非常常用

重点比较：
1. set/map：有序，底层通常是红黑树，查找O(log n)。
2. unordered_set/unordered_map：无序，底层哈希表，平均查找O(1)。
3. 需要有序输出时用map/set。
4. 只关心快速查找和统计时用unordered_map/unordered_set。

练习：
1. 用set去重并输出从小到大的数字。
2. 用map统计每个学生的分数。
3. 用unordered_map统计一篇文章中每个单词出现次数。
4. 用map实现通讯录：姓名 -> 电话。
5. 对自定义Student类型使用set，并定义排序规则。


第4阶段：容器适配器

要学什么：
1. stack
   - push、pop、top、empty、size
   - 后进先出
   - 适合括号匹配、表达式求值、撤销操作

2. queue
   - push、pop、front、back
   - 先进先出
   - 适合排队、BFS

3. priority_queue
   - 默认大根堆
   - top永远是优先级最高的元素
   - greater实现小根堆
   - 自定义比较器

重点：
适配器不是完整容器，它们限制了访问方式，让数据结构更符合特定场景。

练习：
1. 用stack判断括号是否合法。
2. 用queue模拟银行排队叫号。
3. 用priority_queue找出一组数字中最大的前K个。
4. 用priority_queue完成任务优先级调度。


第5阶段：迭代器

要学什么：
1. 迭代器基本写法
   - begin()
   - end()
   - ++it
   - *it

2. 常见迭代器类型
   - iterator
   - const_iterator
   - reverse_iterator

3. 迭代器分类
   - 输入迭代器
   - 输出迭代器
   - 前向迭代器
   - 双向迭代器
   - 随机访问迭代器

4. 不同容器的迭代器能力
   - vector、deque、array：支持随机访问，可以 it + n
   - list、map、set：不支持 it + n，只能++或--

5. 迭代器失效
   - vector扩容会让原来的迭代器、引用、指针失效
   - vector中间erase后，被删除位置及后面的迭代器可能失效
   - list插入删除通常只影响被删除元素
   - map/set插入一般不影响已有迭代器，删除只影响被删元素

练习：
1. 用迭代器遍历vector、list、map。
2. 用reverse_iterator反向输出vector。
3. 安全删除vector中的偶数。
4. 安全删除map中value小于60的元素。


第6阶段：常用算法

要学什么：
1. 查找类
   - find
   - find_if
   - binary_search
   - lower_bound
   - upper_bound
   - equal_range

2. 统计类
   - count
   - count_if
   - all_of
   - any_of
   - none_of

3. 排序类
   - sort
   - stable_sort
   - partial_sort
   - nth_element

4. 修改类
   - copy
   - replace
   - replace_if
   - remove
   - remove_if
   - unique
   - reverse
   - rotate

5. 数值类
   - accumulate
   - inner_product
   - partial_sum

6. 集合类
   - set_union
   - set_intersection
   - set_difference

重点：
1. 算法通常不会直接知道容器是什么，只通过迭代器工作。
2. remove/remove_if不会真正删除容器元素，需要配合erase使用。
   经典写法：
   v.erase(remove(v.begin(), v.end(), x), v.end());

3. lower_bound/upper_bound要求数据已经有序。

练习：
1. 对vector排序并去重。
2. 找出第一个大于80的成绩。
3. 统计及格人数。
4. 删除所有负数。
5. 计算总分和平均分。
6. 求两个有序数组的交集。


第7阶段：函数对象、比较器和lambda

要学什么：
1. 标准函数对象
   - less
   - greater
   - equal_to

2. 自定义比较器
   - 给sort传比较函数
   - 给set/map传比较规则
   - 给priority_queue传比较规则

3. lambda表达式
   - 用lambda排序结构体
   - 用lambda筛选数据
   - 捕获外部变量

重点：
比较器要满足严格弱序：
1. 如果a应该排在b前面，返回true。
2. 如果a和b等价，两个方向都应该返回false。
3. 不要写出逻辑矛盾的比较规则。

练习：
1. 按学生成绩从高到低排序。
2. 成绩相同按姓名字典序排序。
3. 找出所有成绩大于平均分的学生。
4. priority_queue中按任务截止时间排序。


第8阶段：string和常用工具类型

严格来说string不属于早期STL六大组件，但实际开发中必须熟练。

要学什么：
1. string
   - size、empty
   - substr
   - find
   - erase
   - insert
   - replace
   - getline
   - stoi、stol、stod

2. pair
   - pair<int, string>
   - make_pair
   - first、second

3. tuple
   - tuple<int, string, double>
   - get<0>

4. optional
   - 表示可能没有值
   - C++17内容，了解即可

5. variant
   - 一个变量保存多种可能类型
   - C++17内容，了解即可

练习：
1. 读取一行文本并统计单词数量。
2. 使用pair保存坐标。
3. 使用vector<pair<string, int>>保存姓名和成绩。
4. 用string处理逗号分隔的数据。

五、常见错误和注意事项

1. 不要对空容器调用front、back、top。
   调用前先判断empty。

2. map[key]会创建元素。
   如果只想查询，优先使用find。

3. remove不会真正删除元素。
   正确写法：
   v.erase(remove(v.begin(), v.end(), value), v.end());

4. lower_bound必须用于有序区间。
   如果vector没有排序，结果没有意义。

5. sort不能直接用于list。
   list使用自己的成员函数：
   lst.sort();

6. vector扩容会导致迭代器、引用、指针失效。
   如果需要提前插入大量元素，可以先reserve。

7. set里的元素不能直接修改影响排序的字段。
   如果要改，通常先erase，再insert。

8. priority_queue默认是大根堆。
   如果想要小根堆：
   priority_queue<int, vector<int>, greater<int>> pq;

9. 遍历时删除元素要使用erase返回的新迭代器。
   示例：
   for (auto it = v.begin(); it != v.end(); ) {
       if (*it < 0) it = v.erase(it);
       else ++it;
   }

10. 自定义比较器不要写成 <= 或 >=。
    sort比较器通常使用 < 或 >。

七、建议的学习方法

1. 每学一个容器，先问自己三个问题：
   - 它存什么？
   - 它适合什么操作？
   - 它不适合什么操作？

2. 每学一个算法，先问自己三个问题：
   - 它需要什么迭代器？
   - 它是否要求数据有序？
   - 它是否会真正改变容器大小？

3. 每天至少写30分钟代码。
   STL靠看是不够的，必须多敲、多调试、多比较。

4. 建议每个知识点都写一个最小示例。
   例如：
   - vector_demo.cpp
   - map_demo.cpp
   - sort_demo.cpp
   - priority_queue_demo.cpp

5. 每周做一次总结。
   总结内容：
   - 本周学了哪些容器？
   - 每个容器适合什么场景？
   - 哪些API容易写错？
   - 哪些题目还不会？
